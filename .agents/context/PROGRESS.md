## 2026

### April

- 2026-04-27 · _(this PR)_ · feat(recommendations): item-item collaborative filtering with cold-start fallback. New `apps/api/src/recommendations/` (service + controller + spec) implementing cosine on the binary like vectors as two `$queryRaw` SQL calls per seed, aggregated in JS across the user's recent likes. Three endpoints behind a 5-min Redis cache via `CacheService.wrap`: `GET /api/recommendations/me` (authed), `/popular` (public), `/similar/:trackId` (public). Cold start is layered (< 2 likes → popularity → trending; popularity itself falls back to trending when the `Like` table is empty). Resolves heterogeneous track ids (`jm_<jamendoId>` / `demo_t_<id>` / cuid) through `TracksService.byId` so a recommendation list can mix sources. New home-page section on web (`RecommendedSection`) and Mini App (`MiniRecommendedSection`) — picks `/me` or `/popular` based on `localStorage['melodix.token']`, hidden when empty. 8/8 spec tests cover similarity ranking, cold-start fallbacks, popularity ordering, and 404 resolution. ARCHITECTURE / FEATURES / ROADMAP / DECISIONS updated; ADR-0024 added.
- 2026-04-27 · #24 · feat(player): waveform peaks for uploaded tracks. Admin upload + bulk-upload pages decode the audio file client-side via `OfflineAudioContext` (`apps/admin/src/lib/peaks.ts`), downsample to ~200 normalized amplitude buckets, and ship them as a JSON FormData field on `POST /api/admin/tracks`. New nullable `Track.peaks` JSON column with a `normalizePeaks` validator (`apps/api/src/tracks/peaks.util.ts` + spec) reused by every Prisma → `Track` mapper (admin-tracks, tracks, artists, search). New SVG `Waveform` component in `packages/ui` rendered by web `PlayerBar` and Mini App `MiniPlayer` when peaks are present; falls back to the existing `<input type="range">` scrubber for Jamendo / demo tracks. Zero new server-side audio dependencies (no ffmpeg / `audiowaveform`). ARCHITECTURE / FEATURES / ROADMAP / STACK / DECISIONS updated; ADR-0023 added (and ADR-0011 / ADR-0022 unaffected).
- 2026-04-27 · #23 · feat(playlists): drag-to-reorder + per-row remove on Mini App; web reorder upgraded from up/down arrows to drag-and-drop. Mini App: new `EditablePlaylistTracks` + `MiniTracksAuthBoundary` so the playlist page swaps to the editable view only for the owner; `lib/api.ts` gains `reorderPlaylist` + `removePlaylistTrack`. ADR-0011 superseded by ADR-0022.

- 2026-04-26 · #12 · Lyrics view follow-up: surface transient upstream failures correctly. The web `LyricsDrawer` and Mini App `LyricsSheet` now treat a response whose `source === 'none'` (the tag the server uses for upstream timeouts / non-200 / non-404 errors, mirrored by the client `safe()` wrapper) as a retry-able **error** state instead of a genuine "no lyrics" miss.
