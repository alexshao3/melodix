- 2026-04-29 · #37 · fix(aeneas): split `pip install` so `numpy<2` is fully installed before `aeneas==1.7.3.0` runs `setup.py`. Aeneas's `setup.py` does `import numpy` at metadata-generation time, and modern pip's resolver does not make a sibling wheel importable inside the same `pip install` invocation, so the previous single-step install failed with `[ERRO] You must install numpy before installing aeneas` during `docker compose up --build`.
